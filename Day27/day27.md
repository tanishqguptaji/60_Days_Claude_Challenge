# Day 27 – Prior Authorization Story Simulator

## ABTalks 60 Days Claude Challenge

### Project Overview

Today I built an interactive **Prior Authorization Story Simulator**, an educational web application that teaches the U.S. healthcare Prior Authorization (PA) process through an engaging story-driven conversation.

Instead of explaining PA using documentation alone, the simulator walks users through Rahul's healthcare journey while introducing insurance workflows, payer reviews, denials, appeals, and approvals using beginner-friendly conversations.

---

# Objective

Build a single-page interactive story application that explains the complete Prior Authorization workflow through multiple conversation scenes using HTML, Tailwind CSS, and Vanilla JavaScript.

---

# Features Implemented

## Interactive Story Experience

The application follows Rahul's complete healthcare journey across **8 connected scenes**.

### Story Flow

1. Doctor Visit
2. Insurance Roadblock
3. What is Prior Authorization?
4. Insurance Review
5. Prior Authorization Denial
6. Appeal Process
7. Final Approval
8. Key Takeaways

Each scene contains educational conversations and interactive choices that influence the dialogue while maintaining story continuity.

# Technical Highlights

The project was developed while following important implementation constraints:

- Single HTML file
- No frameworks
- No backend
- Dynamic DOM rendering
- Chat bubbles created using createElement()
- appendChild() used for every new conversation
- No use of innerHTML on the chat container
- Mobile responsive layout
