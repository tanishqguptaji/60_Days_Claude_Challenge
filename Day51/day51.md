# Product Discovery & Sprint Planning

## Project Chosen

# VoiceSQL

**Ask your database a question. Out loud. In plain English.**

VoiceSQL is an AI-powered browser application that lets users speak a question about a database using natural language. The application converts voice into text, translates the request into SQL using Claude, safely executes only read-only queries on a sample SQLite database, and displays both the generated SQL and the results.

This project is being developed as my 10-day capstone for the AB Talks 60-Day Claude AI Challenge.

---

# Why I Chose This Project

I wanted to build something that combines multiple AI engineering skills into one polished product.

Instead of creating another chatbot, VoiceSQL demonstrates:

- Speech Recognition
- Prompt Engineering
- LLM-powered SQL Generation
- Database Querying
- Backend API Development
- Full Stack Development
- AI Safety (Read-only SQL execution)

Most importantly, it solves a real problem—allowing non-technical users to interact with databases without writing SQL.

---

# Biggest Scoping Decision

One of the most important decisions was intentionally limiting the project scope.

Instead of supporting every database and advanced AI features, Version 1.0 focuses on:

- One sample SQLite database
- Browser voice input
- Natural language to SQL conversion
- Read-only SQL execution
- SQL transparency
- Clean recruiter-friendly UI

The following features were intentionally excluded:

- Authentication
- User-uploaded databases
- Multi-turn conversations
- Charts and dashboards
- Mobile application
- Paid services

Keeping the scope small makes it realistic to ship a polished product within 10 days.

# Key Learnings

- Product discovery is about solving the right problem before writing code.
- A realistic, well-defined v1.0 is far more valuable than an overly ambitious product.
- A professional PRD becomes the single source of truth for development.
- Breaking the project into daily milestones reduces complexity and keeps development focused.
- Early planning significantly reduces implementation risk during later stages.
- Clear scope boundaries help ensure a polished final product can be shipped on time.
